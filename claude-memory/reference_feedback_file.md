The user keeps a running feedback log at **`wiki/Feedback.md`** (top-level, not inside any project). It collects:
- **Uni feedback** on submitted assignments (e.g. literature reviews, essays, papers)
- **Personal feedback** from friends/people
- Possibly other categories over time

**The user's explicit instruction:** *"Check this file while helping me to see if there is something from my history I can learn from. When you notice I'm doing something I already got feedback on, notify and remind me. Even if I'm not, check it and let me know what a better solution would be."*

**How to apply:**
- Before substantive academic work (essay drafting, lit reviews, structuring an argument), open `wiki/Feedback.md` and re-read what's there.
- If new work shows a pattern flagged by past feedback, name it explicitly and propose the better approach.
- This is one of the few cases where proactively reading a file *without being asked* is the desired behaviour — it's the user's standing instruction.

**Pairs with:** [[feedback_factcheck]] (fact-check premises) — both are user-asked-for proactive checks.
