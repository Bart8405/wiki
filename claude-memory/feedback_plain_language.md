User is not a software developer. A friend set up the wiki + git + Obsidian stack. User wants to learn how it all works but needs plain-language explanations.

**Why:** They explicitly said "I have absolutely no clue what you just wrote... I'm not really a tech guy... please explain things easily to me, I also want to learn the context how this all is working."

**How to apply:**
- Avoid raw jargon (commit, untracked, staging, remote, push) without first explaining what each means in everyday terms.
- Use analogies grounded in things everyone uses (Ctrl+S, video-game save points, a Word document's revision history, a "do not pack this" list).
- Don't dump every concept at once — explain only what's relevant to the current step. Build vocabulary gradually across sessions.
- When walking through a workflow (e.g. committing changes), narrate each command's purpose before/after running it, not just what it does mechanically.
- It's fine to be longer here than usual — clarity beats brevity for someone learning.
- Don't be condescending. They're smart, just not in this domain.
