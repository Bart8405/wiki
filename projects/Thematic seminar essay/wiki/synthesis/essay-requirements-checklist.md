# Essay Requirements & Checklist

*Structured rubric from the final seminar session, plus a checklist to track whether the essay meets each requirement. Open this file alongside the draft while writing.*

> **Use this file like a marking sheet.** Tick `[x]` as each requirement is satisfied. Anything still `[ ]` when you're "done" is a gap — fix it before submission.

---

## 0. Overarching principles (apply everywhere)

These are not section-specific — they govern the whole essay.

| # | Principle | Translation |
|---|---|---|
| 0a | **Frontload everything** | Don't save your argument for the conclusion. The reader should know what you think by paragraph 2. |
| 0b | **Not a crime novel** | No suspense, no reveals. Tell the reader what's coming and then deliver it. |
| 0c | **Explain carefully** | Define terms on first use. Don't assume the reader has read the same things you have. |
| 0d | **Have a clear argument — and state it** | One sentence the reader can repeat. State it explicitly. Repeat it. |
| 0e | **Everything is there to make the argument** | If a sentence, paragraph, or section doesn't push the argument forward, cut it. |
| 0f | **Engage with the literature** | Use the literature actively — not as scaffolding, not as decoration. |

**Checklist:**
- [ ] My argument is one clear sentence I can recite
- [ ] The argument appears in the introduction, not just the conclusion
- [ ] I have not written a "reveal" — every section is doing visible work for the argument
- [ ] Every technical term is defined where it first appears
- [ ] No section is in the essay just because it "feels related" — each has a job

## 1. Introduction

**What the introduction must do** (4 jobs):
1. **Puzzle** — pose the problem clearly
2. **Question** — what specifically am I asking?
3. **Argument** — what is my answer?
4. **Roadmap** — how is the essay going to deliver the answer?

**Style requirements:**
- **Reference a lot.** The intro itself should be heavily cited — this is where you establish that you're in a real debate.
- **Forward-link to specific sections.** *"As will be discussed in Section 2…"*, *"Section 3 returns to…"* This is anti-crime-novel signalling.
- **Optional but encouraged:** methodology paragraph — case selection, data choices, scope justification (see §4 below).

**Checklist:**
- [ ] Opens with a **puzzle** (not "carbon pricing is important and global…")
- [ ] **Question** stated explicitly (one sentence, ending in a question mark or "this essay asks…")
- [ ] **Argument** stated up front (frontloaded, not "as I will show below")
- [ ] **Roadmap** previews each section ("Section 1 does X; Section 2 does Y; Section 3 does Z")
- [ ] **Many references** woven through the intro paragraphs
- [ ] At least one **forward link** to a specific later section
- [ ] Methodology paragraph included OR justified to be in §4

## 2. Literature review

**Length:** roughly **one-third** of the essay.

**What it must NOT be:** a list. Not "Author A says X. Author B says Y. Author C says Z."

**What it MUST do:**
- Identify the **positions** and **(dis)agreements** in the literature
- **Mobilize** the literature to build *your* argument
- For every claim you make: "I think X — based on what?" Each load-bearing claim is anchored to a source
- Demonstrate **awareness of the debates** *and use them*
- Show **where you stand** within the debates

**The seminar instruction in one sentence:** *"You are not being asked to do something new — rather demonstrate that you are aware of what the debates are and use them."*

**The supervisor's literature-review feedback applies here too** (from [wiki/Feedback.md](../../../../Feedback.md)):
> *"Cover less ground but in greater detail. Each point should be stated, supported, illustrated, and assessed. This will help the text move beyond an overview of existing approaches and towards a critical engagement with the literature in order to develop its own analytical framework."*

This is the **biggest known weak spot** based on past feedback. Read this line again before drafting the lit review.

**Checklist:**
- [ ] Lit review is approximately **1/3 of total word count**
- [ ] Organized **by debate / position**, not paper-by-paper
- [ ] Identifies what scholars **agree** on
- [ ] Identifies what scholars **disagree** on
- [ ] Each load-bearing claim is **anchored to a source**
- [ ] Reader can see **where I stand** in the debate (not just "here is what others say")
- [ ] Each point follows the four-move pattern: **stated → supported → illustrated → assessed**
- [ ] I have **cut something** — covered less, in more depth (per the LR feedback)
- [ ] My own argument is **visibly being built** out of these debates

## 3. Body of the essay

**Structure:** **Funnel** — start general, move to specific.

**The seminar instruction:** *"How does a broad debate play out in a specific example?"* That's the move. The lit review establishes the broad debate; the body shows it operating on the Indonesia case.

**What the body must do:**
- Apply the analytical framework you built in the lit review
- Use specific empirical evidence
- Address counter-arguments explicitly
- Build cumulatively toward the argument stated in the intro

**Checklist:**
- [ ] Opens at the **general** level (frameworks, debates) — funnel top
- [ ] Narrows to **specific** Indonesia evidence — funnel bottom
- [ ] Each argument is **supported by evidence**
- [ ] Counter-arguments are **named and addressed** (not avoided)
- [ ] Each section visibly **continues building the argument** from the intro
- [ ] No section reads like "additional information" — every one is load-bearing

---

## 4. Methodology

**Important seminar note:** *"I don't have to have a separate methodology section. I should have a paragraph in the introduction about it."*

**What the methodology paragraph must do:**
- Justify **case selection**: why Indonesia (not Vietnam, not Brazil, not "developing countries in general")?
- Justify **data / source choices**: why these reports, this empirical data?
- Justify **scope choices**: *why am I doing X and not Y?* (e.g., why focus on the ETS-vs-tax sequencing and not REDD+? Why political economy and not formal modelling?)
- Acknowledge: **the choices were not random** — they were made for reasons.

**Checklist:**
- [ ] Case selection (Indonesia) is **explicitly justified** somewhere
- [ ] Source / data choices are **defended**, not just listed
- [ ] **Scope exclusions** ("I'm not covering X because…") are explicit
- [ ] The methodology paragraph appears in the **introduction** (or there's a clear methodology paragraph somewhere visible)

---

## 5. Conclusion

**What the conclusion must do:**
- **Link back** to specific other parts of the text. *"As discussed in Section 2…"*, *"The literature reviewed in Section 1 implies…"*
- **Restate the argument** — same argument as the intro, not a new one
- Briefly point to **implications** (what this means beyond the case)

**The seminar instruction:** the conclusion should *confirm* what the reader already knows from the introduction. Because of frontloading, the conclusion is a destination, not a surprise.

**Checklist:**
- [ ] Conclusion **restates** the argument (same one as intro)
- [ ] Does **NOT** introduce new arguments or evidence
- [ ] Contains **backward links** to specific sections
- [ ] Closes the loop on the **puzzle** from the intro
- [ ] Suggests **implications** beyond the Indonesia case

## 6. The argument itself (the load-bearing requirement)

The single most important thing in the essay. The seminar explicitly returned to this multiple times.

**What the argument must be:**
- **Clear** — one sentence
- **Stated up front** — visible in the intro
- **Yours** — your own position, not a survey
- **Built from the literature** — defended with sources, not just asserted

**How to build the argument (the seminar's three-step recipe):**
1. Identify the **few schools / debates** that bear on the question
2. Locate **where you stand** within those debates
3. **Apply your position** to the case (Indonesia)

**Additional move:** combine **multiple debates** together. *"Putting multiple debates together and approaching the question."* The strongest essays don't sit in one debate — they bridge two or three.

**The "Pose a problem and formulate an argument against it":** Pick the obvious / lazy answer to your question and argue against it explicitly. That's how a reader knows your argument is non-trivial.

**Checklist:**
- [ ] I can state my argument in **ONE sentence**
- [ ] The argument is **visible in the intro** (not just the conclusion)
- [ ] My **position within the debates** is identifiable to the reader
- [ ] I am engaging **multiple debates**, not just one
- [ ] The Indonesia case is **proving or illustrating** the argument, not just providing background
- [ ] I have **posed a problem and argued against** the obvious / lazy answer
- [ ] If a reader stops reading at any point, they could still tell you what I'm arguing

## 7. Process notes (not requirements, but seminar advice)

- **Write more, read less.** Set aside an hour where you do nothing else but write. The reading is mostly done — the bottleneck is now writing.
- **References count toward word count; bibliography does not** *(based on the seminar note "includes references but not bibliography"; verify this against the assignment brief if uncertain).*

## Master checklist (high-level — tick before submitting)

- [ ] **Argument:** can be stated in one sentence; appears in intro; defended throughout
- [ ] **Introduction:** puzzle + question + argument + roadmap + many references + ≥1 forward link
- [ ] **Literature review:** ~1/3 of words; organized by debate not paper; each claim sourced; cuts depth over breadth
- [ ] **Body:** funnel structure (general → specific); counter-arguments named; empirically anchored
- [ ] **Methodology:** case selection, data choices, scope exclusions all defended (likely in intro paragraph)
- [ ] **Conclusion:** restates not reveals; backward links; closes the puzzle
- [ ] **Style:** frontloaded, not a crime novel; everything serves the argument; no padding
- [ ] **Past feedback applied:** less ground, more depth, critical engagement (Lit Review feedback in `Feedback.md`)

- ⚠️ **Counter-arguments named and addressed** — `argument-logic.md` did a stress-test against 5 objections, but those need to be visible in the essay itself, not just in the prep notes. At least one paragraph in §1 or §3 should say "the standard counter-argument is X; my response is Y."

The lit review feedback in [Feedback.md](../../../../Feedback.md) says: *"cover less ground but in greater detail."* The current §3 ("What Indonesia should do") tries to cover **five recommendation moves + CBAM** in 1,000 words.
