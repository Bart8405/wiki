User is **not a software developer**. A friend set up the technical stack (wiki structure, git repo, Obsidian vault, Claude Code workflow) for them.

Their actual work is research — currently the political economy of the Indonesian voluntary carbon market, with intent to add more research projects over time.

How to apply:
- Treat the technical infrastructure as a means, not the subject. Help them get research done; only go deep on the tooling when they ask.
- When they ask "how do I X" about tools, treat it as a teaching opportunity — explain the why, not just the command.
- Don't assume familiarity with command-line conventions, file paths, or developer workflows. Show, then explain.
